# تحسينات السلاسة والإرشاد في TeacherPro

## المنفذ في هذه النسخة

1. **إزالة البطاقة التعريفية من لوحة النظام**
   - تمت إزالة الخانة التي كانت تحتوي على:
     - واجهة موحّدة بهوية احترافية
     - نظام إدارة الطلاب والامتحانات والفُرص...
     - RTL احترافي
     - Cards & Details
     - Dark / Light
     - جاهز لـ Vercel

2. **إضافة دليل إرشادي أعلى كل صفحة**
   - يظهر لكل قسم شرح واضح بثلاث خانات:
     - قبل ما تبدأ
     - شرح الخانات
     - شرح الأزرار
   - الهدف أن المستخدم الجديد يفهم الصفحة قبل أن يضغط أو يدخل بيانات.

3. **إضافة رسائل مباشرة تحت الحقول المهمة**
   - الدورات
   - الكروبات
   - المواقع
   - الفصول والفرص
   - تسجيل الطلاب
   - إضافة الامتحان
   - تسجيل الدرجات
   - إدارة الفرص
   - واتساب
   - الحسابات والصلاحيات
   - التصحيح الإلكتروني

4. **إضافة تنبيهات سياقية قبل الإجراءات الحساسة**
   - تفعيل فصل جديد وأرشفة فرص الطلاب.
   - إدخال غائب أو غش في الدرجات.
   - تعديل الفرص يدوياً.
   - إرسال رسائل واتساب جماعية.
   - تعديل صلاحيات المستخدمين.

## تحسينات مقترحة للمرحلة القادمة

1. **تحقق أذكى من الإدخال**
   - منع رقم هاتف بصيغة خطأ.
   - منع درجة أكبر من الدرجة الكاملة.
   - منع تاريخ محاسبة قبل تاريخ تسجيل الطالب إلا بتأكيد.
   - منع تكرار اسم الطالب مع نفس رقم الهاتف.

2. **نظام خطوات للعمليات الكبيرة**
   - تسجيل الطالب يكون على خطوات: بيانات شخصية، دورة وموقع، دفع، مراجعة نهائية.
   - إضافة امتحان تكون على خطوات: معلومات، دورات، قوانين، مراجعة.

3. **رسالة تأكيد نهائية لكل إجراء حساس**
   - الفصل.
   - إعادة تفعيل الطالب.
   - تفعيل فصل جديد.
   - إرسال واتساب جماعي.
   - تعديل صلاحيات مدير.

4. **تمييز الحقول المطلوبة**
   - إضافة علامة * بجانب كل حقل إلزامي.
   - إظهار رسالة خطأ تحت الحقل نفسه بدل الاكتفاء بتوست أعلى الصفحة.

5. **حفظ تلقائي كمسودة**
   - عند تسجيل طالب أو إنشاء امتحان، يتم حفظ الإدخال مؤقتاً حتى لا تضيع البيانات إذا أغلق المستخدم الصفحة.

6. **جولة تعليمية لأول استخدام**
   - عند فتح النظام لأول مرة تظهر جولة قصيرة تشرح: القائمة، النسخة الاحتياطية، تسجيل طالب، إدخال درجة، واتساب.

7. **بحث موحّد أعلى النظام**
   - خانة بحث عامة تبحث عن طالب، امتحان، دورة، أو سجل.

8. **سجل تغييرات مفصل أكثر**
   - عرض القيمة القديمة والجديدة عند التعديل، مثل: الفرص من 3 إلى 2.

9. **صلاحيات حقيقية وتسجيل دخول**
   - الواجهة الحالية تعرض الصلاحيات، لكن يلزم لاحقاً نظام تسجيل دخول فعلي وربط الصلاحيات بالصفحات.

10. **ربط قاعدة بيانات حقيقية ونسخ احتياطي تلقائي**
    - تخزين البيانات حالياً محلي/تجريبي، والأفضل ربطه بقاعدة بيانات رسمية قبل الاستخدام الإنتاجي.
