# ملاحظات نشر TeacherPro على Vercel

## ما الذي يعالجه هذا التحديث؟

- تشغيل `prisma generate` أثناء البناء حتى لا تفشل وظائف API بسبب Prisma Client قديم أو غير مولّد.
- استخدام `npm ci` على Vercel للاعتماد على `package-lock.json` بدل تثبيت غير حتمي.
- توفير قيمة `DATABASE_URL` احتياطية وقت البناء فقط حتى لا يفشل البناء عند غياب المتغير، مع بقاء قاعدة البيانات الحقيقية مطلوبة لتشغيل API في الإنتاج.
- إجبار مسارات API على Runtime من نوع Node.js وعلى تنفيذ ديناميكي لأن Prisma لا يعمل كمسار Static أو Edge.
- تثبيت إصدار Node المناسب للمشروع عبر `engines`.

## خطوات مهمة بعد تطبيق الباتش

1. في Vercel، أضف متغير البيئة `DATABASE_URL` في Production وPreview.
2. بعد ربط قاعدة البيانات، شغّل migrations مرة واحدة من بيئة تملك نفس متغيرات قاعدة البيانات:

```bash
npm run db:deploy
```

3. ارفع المشروع إلى GitHub ثم اربطه بـ Vercel، أو أعد النشر إذا كان مربوطًا مسبقًا.

> البناء يمكن أن يمر بدون `DATABASE_URL` بفضل القيمة الاحتياطية، لكن وظائف API التي تقرأ/تكتب البيانات تحتاج قاعدة PostgreSQL حقيقية في وقت التشغيل.
