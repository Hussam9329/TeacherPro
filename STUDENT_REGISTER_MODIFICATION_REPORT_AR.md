# تقرير تعديل صفحة تسجيل الطالب

## ملخص التنفيذ
تم تطبيق محتوى ومنطق ملف HTML المرفق على صفحة تسجيل الطالب داخل مشروع Next.js/React بدل إدراج HTML خام، مع الحفاظ على بنية المشروع الحالية وربطها بـ Zustand وAPI وPrisma.

## الملفات التي تم تعديلها

1. `src/components/teacher-pro/student-register.tsx`
   - استبدال واجهة التسجيل القديمة بتصميم قريب من ملف HTML المرفق.
   - إضافة حقول: اسم المدرسة، نوع الدورة، تفاصيل الدورة الخاصة، رقم الوصل، تسلسل الكود، المبلغ الكلي، المبلغ المدفوع، المتبقي، فترة السماح، والجنس.
   - جعل نوع الدورة الافتراضي `عامة` مثل ملف HTML.
   - تثبيت الموقع الرئيسي تلقائياً على `بغداد` عند اختيار دورة `خاصة`.
   - عرض المواقع الفرعية الخاصة فقط: `المنصور`، `زيونة`، `البنوك`.
   - حساب المبلغ المتبقي مباشرة من: `المبلغ الكلي - المبلغ المدفوع`.
   - التحقق من أن المبلغ المدفوع لا يتجاوز المبلغ الكلي.
   - تحويل أرقام الهاتف والمبالغ إلى أرقام لاتينية وتنظيف المدخلات.

   أهم المقاطع:
   - نموذج البيانات والحقول الجديدة: الأسطر 27-45.
   - القيم الافتراضية للنموذج: الأسطر 51-70.
   - منطق المواقع حسب نوع الدورة: الأسطر 110-119.
   - التحقق من الحقول والمبالغ: الأسطر 162-207.
   - حفظ الطالب وإرسال الحقول الجديدة إلى المتجر: الأسطر 209-245.
   - قسم بيانات الدورة الخاصة والأقساط: الأسطر 350-426.
   - قسم المواقع والكروب الإلكتروني: الأسطر 432-480.
   - فترة السماح والجنس: الأسطر 482-511.

2. `prisma/schema.prisma`
   - إضافة حقل `school` إلى جدول الطالب.
   - إضافة `totalAmount` و `paidAmount` لتخزين بيانات الأقساط الخاصة.

   الكود المضاف داخل `Student`:

   ```prisma
   school          String   @default("")
   totalAmount     Int      @default(0)
   paidAmount      Int      @default(0)
   ```

3. `src/app/api/students/route.ts`
   - إضافة تحقق خلفي من `school`.
   - جعل `telegram` غير إلزامي لأن ملف HTML لا يحتوي عليه.
   - إضافة تحقق خلفي من مبالغ الدورة الخاصة.
   - منع حفظ المبلغ المدفوع إذا كان أكبر من المبلغ الكلي.
   - جعل الموقع الفرعي مطلوباً فقط عند الحاجة: دورة خاصة أو محافظات.
   - حفظ `school`, `totalAmount`, `paidAmount` في قاعدة البيانات.

   أهم المقاطع:
   - الحقول المطلوبة: الأسطر 20-32.
   - شرط الموقع الفرعي: الأسطر 34-37.
   - شروط مبالغ الدورة الخاصة: الأسطر 39-45.
   - حفظ الحقول الجديدة: الأسطر 54-72.

4. `src/lib/teacher-store.ts`
   - تحديث نوع `Student` لدعم `school`, `totalAmount`, `paidAmount`.
   - تطبيع البيانات القادمة من السيرفر للحقول الجديدة.

   أهم المقاطع:

   - تطبيع بيانات الطلاب من السيرفر: الأسطر 744-754.
   - تنظيف المدرسة عند إضافة/تعديل طالب: الأسطر 1066-1104.

5. `src/components/teacher-pro/student-registry.tsx`

   أهم المقاطع:

   - تعبئة نموذج التعديل من الطالب: الأسطر 59-76.
   - تحقق التعديل من المدرسة والمبالغ: الأسطر 184-231.
   - حفظ الحقول الجديدة في التعديل: الأسطر 233-265.
   - عرض المدرسة والأقساط في بطاقة الطالب: الأسطر 420-472.

6. `src/lib/student-utils.ts`
   - تعديل فترة السماح لتقبل الرقم `0` كما في ملف HTML، بدل 1-30 فقط.

   ```ts
   return /^(?:[0-9]|[12][0-9]|30)$/.test(toLatinDigits(value).trim());
   ```

## ملاحظات تشغيل مهمة

بعد رفع المشروع المعدل على بيئة التشغيل، يجب تنفيذ أوامر Prisma لتحديث قاعدة البيانات بسبب إضافة أعمدة جديدة:

```bash
npm install
npm run db:generate
npm run db:push
npm run build
```

أو مباشرة:

```bash
npx prisma generate
npx prisma db push
```

## فحص سريع تم تنفيذه

- تم تثبيت الاعتماديات محلياً للفحص.
- تم تشغيل `npm run lint`، وبقيت أخطاء ESLint قديمة في ملفات غير مرتبطة بالتعديل الحالي:
  - `src/app/page.tsx`
  - `src/components/teacher-pro/exam-new.tsx`
  - `src/components/teacher-pro/layout.tsx`
- بعد تعديل ملفات التسجيل، لم تعد صفحة تسجيل الطالب أو سجل الطلاب تظهر ضمن أخطاء ESLint الرئيسية.
- تم تشغيل `npx tsc --noEmit`، وتوقف فقط لأن `PrismaClient` غير مولّد في بيئة الفحص الحالية، حيث فشل `prisma generate` بسبب عدم القدرة على تحميل binaries من `binaries.prisma.sh` داخل البيئة المعزولة. هذا لا يدل على خطأ في الكود؛ يجب تشغيل `prisma generate` على جهاز متصل بالإنترنت أو بيئة لديها Prisma engines.
